# 가사 멜로디 표기 (lyric-melody)

노래 가사의 글자 하나하나 위에 **계단식 멜로디 선**과 **바이브레이션 숫자**를 그려 넣는 웹 편집기입니다.

## 표기 규칙

- 음높이는 상대적입니다. 음이 바뀌면 고정 크기의 계단이 한 칸 생깁니다.
- 한 글자에 음이 1개면 아무 표시가 없고, 2음 이상(연음)일 때만 계단 선이 그려집니다.
- 올라가는 계단, 내려가는 계단 모두 가능합니다.
- 바이브레이션은 해당 음 위에 숫자로만 표시합니다. 계단 없이 숫자만 단독으로도 붙일 수 있습니다.

## 사용법

1. 상단에 가사를 입력하고 **[가사 적용]** (줄바꿈으로 여러 줄 구분)
2. 글자를 클릭한 뒤 **`↑` `↓`** 를 누르면 계단이 쌓입니다 (마지막 음에서 자동으로 연음 추가)
3. **`1~9`** 로 선택한 음에 바이브레이션 숫자 지정, **`0`** 으로 지우기, **`Delete`** 로 음 삭제
4. **자주 쓰는 패턴** 버튼으로 꺾기 패턴을 한 번에 적용하고, [＋ 현재 패턴 저장]으로 나만의 패턴을 추가
5. **[인쇄]** 로 악보만 깔끔하게 출력, 하단 JSON으로 내보내기/가져오기
6. **[타임라인] → [● MR 녹음]** 으로 원곡은 끄고 MR만 들으며 마이크 녹음. 기본은 MR+목소리 합본이며, 체크를 끄면 목소리만 녹음합니다.

악보, 사용자 패턴, 원곡/MR은 로그인한 Google 계정별로 Firebase에 자동 저장되며, 브라우저에도 복구용 사본을 보관합니다.
다른 컴퓨터에서는 반드시 같은 Google 계정으로 로그인해야 같은 작업을 불러옵니다.
녹음 결과는 기존 원곡/MR을 덮어쓰거나 클라우드에 자동 업로드하지 않으므로, 완료 후 **[녹음 저장]** 으로 기기에 내려받아 보관합니다.

## 실행

운영 주소: [https://lyric-melody.vercel.app/](https://lyric-melody.vercel.app/)

빌드 과정이 없는 정적 페이지입니다. 로컬 개발 시에는 파일을 직접 열지 말고 HTTP 서버로 실행합니다.

```bash
python3 -m http.server 8000
```

그런 다음 [http://localhost:8000](http://localhost:8000)으로 접속합니다. `file://`로 `index.html`을 직접 열면 Firebase Google 로그인이 허용되지 않습니다.

## Firebase 보안 규칙

Firestore와 Storage는 로그인한 사용자가 자신의 `users/{uid}` 경로만 읽고 쓸 수 있도록 규칙 파일을 포함합니다.
Firebase CLI로 규칙을 배포할 때는 다음 명령을 사용합니다.

```bash
firebase deploy --only firestore:rules,storage
```

Storage 오디오를 브라우저에서 다운로드하려면 버킷 CORS도 한 번 적용합니다.

```bash
gcloud storage buckets update gs://lyric-d92bb.firebasestorage.app \
  --cors-file=storage.cors.json
```
